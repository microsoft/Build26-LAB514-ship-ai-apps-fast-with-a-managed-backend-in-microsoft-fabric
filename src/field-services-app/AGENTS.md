# AGENTS.md

## Project

React + TypeScript (Vite) work-order app backed by **Project Rayfin** with Radix-based UI components styled with Tailwind CSS.
Demonstrates Service Pro profile and work-order management with Fabric Entra authentication enabled for deployed Fabric environments.
Auth is fully integrated: local dev uses Rayfin email/password auth, production uses Fabric Entra SSO.

## Quickstart

- Deploy to Fabric: `npx rayfin up`
- Frontend against Fabric backend: `npm run dev:fabric`
- (Alternative) Backend locally: `npm run rayfin:dev`
- (Alternative) Apply DB schema: `npm run rayfin:db`
- (Alternative) Frontend locally: `npm run dev`

## Repo map

- `src/` – React UI
- `src/components/ui/` – Radix-based UI components (shadcn)
- `src/components/` – App components (AuthPage and UI components)
- `src/hooks/` – React hooks (useFieldService, AuthContext)
- `src/pages/` – Page components (Dashboard, AuthCallback)
- `src/services/` – Service layer (ServiceContainer, Rayfin services, auth services)
- `rayfin/rayfin.yml` – Rayfin configuration (auth and data enabled)
- `rayfin/data/` – Entities (export from `rayfin/data/schema.ts`)

## UI (Radix-based shadcn/ui)

This project uses **Radix-based UI components** styled with **Tailwind CSS v4**.

- UI components live in `src/components/ui/`
- Utility helper lives in `src/lib/utils.ts` (`cn`)
- Add a component: `npx shadcn@latest add <component>`

Key config/files:

- Global styles + Tailwind import: `src/styles/theme.css`
- Path alias `@/*` → `src/*`: `tsconfig.json`
- Vite plugins + alias: `vite.config.ts`
- shadcn registry/config: `components.json`

## Data model changes

- Edit/add entities in `rayfin/data/` and export them from `rayfin/data/schema.ts`.
- After entity/relationship changes, apply DB migrations: `npm run rayfin:db` (run while `npm run rayfin:dev` is active).

## Architecture

### Service Layer

- **ServiceContainer**: Singleton that initializes and provides access to services; auto-detects local vs. production and selects the appropriate auth service
- **RayfinClientService**: Manages the RayfinClient singleton instance
- **RayfinFieldService**: Service Pro and work-order CRUD via DataApi fluent interface

### Auth Service Layer

- **RayfinAuthService**: Fabric Entra authentication operations (production)
- **RayfinFabricAuthService**: Delegates to `@microsoft/rayfin-auth-provider-fabric` SDK for Entra flows
- **RayfinPasswordAuthService**: Local development email/password authentication
- **AuthContext**: React context for auth state management

### Routing

- `/auth` – Public auth page (Fabric sign-in or email/password sign-in; redirects to `/` if authenticated)
- `/auth/callback` – Fabric Entra OAuth callback
- `/` – Protected service pro dashboard (redirects to `/auth` if not authenticated)
- `/manager/` – Protected manager dashboard (only accessible by direct URL)
- `/_admin/` – Hidden protected DB data tools page for seed/reset operations

### Key files

- `rayfin/data/ServicePro.ts` – Service Pro profile entity
- `rayfin/data/WorkOrder.ts` – Work-order entity with optional service pro relationship
- `src/services/ServiceContainer.ts` – Service initialization with auth-mode detection
- `src/hooks/useFieldService.ts` – Service Pro/profile/work-order operations hook
- `src/pages/Dashboard.tsx` – Service Pro and manager dashboards
- `src/pages/AdminPage.tsx` – Hidden seed/reset admin data page

## Environment

- `.env` – Default Fabric API configuration
- `.env.local` – Local overrides (git-ignored, generated by `rayfin env --framework vite`)
- `rayfin/.env` – runtime + deployment env (written by `rayfin dev`/`rayfin up`)
- `rayfin/.deployments.json` – registry of all Fabric workspace deployments (written by `rayfin up`)

## Code conventions

- TypeScript strict mode; prefer small functional React components.
- Keep UI logic in `src/`; keep data model code in `rayfin/data/`.
- Services follow the singleton pattern via `ServiceContainer`.
- `ServiceContainer` auto-selects `RayfinPasswordAuthService` for localhost backends and `RayfinAuthService` for production.

## Generated files (do not edit)

- `rayfin/.temp/` – Rayfin/DAB config generation output
- `dist/` – Vite build output
- `.env.local` – Local environment overrides (git-ignored, script-managed)
